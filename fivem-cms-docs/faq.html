よくある質問 (FAQ)

FiveM CMSに関するよくある質問と回答

一般的な質問

FiveM CMSとは何ですか？

FiveM CMSは、FiveMサーバーで使用するデータ（Items、Vehicles、Jobsなど）をブラウザ上で管理し、Luaファイルとして出力できるWebアプリケーションです。shared.luaを直接編集する手間を省き、チームでの共同作業を可能にします。

無料プランでは何ができますか？

無料プランでは以下の機能を利用できます：

  • Itemsテーブルの管理（追加、編集、削除）
  • Luaテーブル形式でのインポート
  • QBCore、ESX、Qbox形式でのエクスポート

カスタムスキーマ、カスタムテンプレート、チームメンバーの追加には有料プランが必要です。

対応しているフレームワークは？

標準で以下のフレームワークに対応しています：

  • QBCore (qb-core, qb-inventory)
  • ESX (es_extended)
  • Qbox (Qbox Project)

カスタムテンプレート機能を使えば、ox_inventory、jg-advancedgarages、eco-craftingなど、任意のスクリプト形式に出力できます。

インポート・エクスポート

既存のshared.luaからデータを移行できますか？

はい！Luaインポート機能を使えば、既存のQBShared.ItemsやESX.Itemsをコピー＆ペーストでインポートできます。フィールドマッピングのプリセットも用意されており、異なる形式のデータも簡単に取り込めます。

ox_inventoryに対応していますか？

標準のエクスポート形式には含まれていませんが、カスタムテンプレートを作成することでox_inventory形式で出力できます。インポート時も「ox_inventory」プリセットを使用できます。

エクスポートしたファイルはどこに配置すればいいですか？

フレームワークとテーブルによって異なります。例：

  • QBCore Items: qb-core/shared/items.lua
  • QBCore Vehicles: qb-core/shared/vehicles.lua
  • ox_inventory: ox_inventory/data/items.lua

配置後はサーバーを再起動（またはリソースを再起動）してください。

チーム・コラボレーション

チームメンバーを招待するには？

ダッシュボードの「Members」→「招待」からメールアドレスを入力して招待できます。相手が既にFiveM CMSに登録している場合は即座に追加され、未登録の場合は招待リンクが生成されます。

招待リンクの有効期限は？

招待リンクは7日間有効です。期限が切れた場合は、再度招待してください。

メンバーの権限を変更できますか？

はい。Owner（オーナー）とAdmin（管理者）はメンバーの役割を変更できます。役割にはOwner、Admin、Member、Viewerの4種類があり、それぞれ権限が異なります。

データ・セキュリティ

データのバックアップはありますか？

データはサーバーに保存されており、いつでもJSON形式やLua形式でエクスポートできます。重要なデータは定期的にエクスポートして保存しておくことをおすすめします。

データは他のチームに見られますか？

いいえ。各チームのデータは完全に分離されており、他のチームからはアクセスできません。

料金・支払い

無料プランに期限はありますか？

いいえ、無料プランは永久に無料です。時間制限や試用期間はありません。

プランをいつでも変更できますか？

はい。いつでもアップグレードまたはダウングレードできます。アップグレードは即座に適用され、ダウングレードは次の請求サイクルから適用されます。

返金ポリシーは？

初回購入から7日以内であれば全額返金いたします。お問い合わせください。

トラブルシューティング

ログインできません

以下を確認してください：

  • メールアドレスとパスワードが正しいか
  • Caps Lockがオフになっているか
  • ブラウザのキャッシュをクリアしてみる

それでも解決しない場合は、パスワードリセットをお試しください。

インポートがうまくいきません

以下を確認してください：

  • 正しい形式（Lua/SQL/JSON）を選択しているか
  • Luaの場合、QBShared.Items = { ... } の形式になっているか
  • プリセットを使用して、フィールドが正しくマッピングされているか

エラーメッセージが表示された場合は、その内容をもとに修正してください。

エクスポートしたファイルがサーバーで動作しません

以下を確認してください：

  • 正しいフレームワーク（QBCore/ESX/Qbox）を選択しているか
  • ファイルを正しいパスに配置しているか
  • サーバーを再起動（またはリソースをリスタート）したか
  • FiveMサーバーのコンソールにエラーが表示されていないか
